# Standby — 产品理念说明书

> 在 AI 时代，一切都可伪造，唯有共鸣无法伪造。

---

## 一、核心命题

**AI 无法伪造的，是你真正的感受。**

当一切都可以被伪造，真实感受就成了最稀缺的资源。

Standby 做两件事：让人敢于表达真实感受，让真实感受成为可理解的信号。


### 让人重新敢于表达真实感受

现有平台贩卖注意力，数据告诉供给侧的是「用户被什么刺激吸引」，而不是「用户真正需要什么」。Standby 不贩卖注意力——它是一个安静的空间，让真实的感受自然浮现。用户因心物而触动，因共鸣而连接，因安心而表达。

### 让真实感受成为可理解的信号

用户的真实表达在加密状态下聚合为群体共识，为供给侧提供「用户真正需要什么」的答案。不是行为数据，不是点击数据，而是感受数据——这个时代最稀缺、最真实、最不可伪造的数据。

**这两件事互为飞轮：** 用户越真实，平台价值越高；平台价值越高，供给侧越愿意付费；供给侧越精准，用户获得的供给越好。没有一方需要牺牲另一方来获益。

## 二、时代观察

### 2.1 技术悖论

```
技术能力 ↑  →  AI 可以生成一切  →  真假难辨  →  信任 ↓
效率工具 ↑  →  流量至上逻辑    →  对立/虚伪 ↑  →  真实表达 ↓
```

结果：人们有能力表达，但不敢表达；有渠道交流，但不愿交流。

### 2.2 AI 时代的两个选择

- 大多数产品选择用 AI 来**生成更多内容**（AI 写作、AI 绘画、AI 视频）→ 内容通胀
- Standby 选择用 AI 来**理解已有内容** → 发现人与人之间隐藏的共鸣

**AI 不应该是人与人之间的隔阂，而应该是桥梁。**

### 2.3 AI 的边界

AI 可以写一首关于孤独的诗，但它不会真的孤独。
AI 可以生成一张夕阳的照片，但它不会真的被夕阳触动。
AI 可以模拟同理心，但它不会真的理解你的痛。

**AI 时代最稀缺的资源是：真实的人类感受。Standby 是感受的空间，不是内容的平台。**

## 三、产品本质

### 3.1 Standby 是什么？

不是社交平台（不追求关系数量）
不是内容平台（不追求内容消费）
不是通讯工具（不追求即时通讯）

**是：让人重新敢于表达真实自我的安全空间。**

### 3.2 一句话定位

**别的产品是「人→内容→人」，Standby 是「人→事物→感受→人」。中间多了一个「事物」心物，一切都不一样了。**

### 3.3 与主流平台的本质区别

| 平台 | 核心问题 | 优化目标 | 社交模型 |
|------|---------|---------|---------|
| 微信 | 你认识谁 | 关系维护 | 好友 + 朋友圈 |
| 抖音 | 你看什么内容 | 参与度 | 内容 + 算法 |
| 微博 | 你想让别人知道什么 | 传播力 | 广播 + 评论区 |
| 小红书 | 你想成为什么样的人 | 生活方式 | 种草 + 分享 |
| **Standby** | **你感受到了什么** | **共鸣** | **心物 + 感受链** |

## 四、设计原则

### 4.1 核心设计原则

| 维度 | Standby 的选择 | 对标现状 |
|------|--------------|---------|
| **内容逻辑** | 真实 > 流量 | 流量 > 真实 |
| **AI 角色** | 守护者 + 织网者 | 助推者（放大内容、优化传播） |
| **交互形态** | 无感的、融入生活的 | 刻意的、打开 App 才存在的 |
| **设备关系** | 系统级输入，不分设备 | App 级隔离 |
| **关系建立** | 先共享体验，再发现人 | 先关注人，再看内容 |

**好的人与人之间的交流是无感的、真诚的、积极的。**

### 4.2 产品哲学

1. **心物驱动，非内容驱动** — 人→事物→感受→人，不是人→内容→流量→人
2. **感受链，非评论区** — 表达本身就是新的心物，形成链式反应
3. **匿名架构，安全优先** — 攻击在架构层面就难以组织
4. **共鸣过滤，非算法推荐** — 不是「你可能喜欢」，而是「你们确实有共鸣」
5. **深度优先，非正负优先** — 深度表达优先于表面表达
6. **沉默表达，非公开表演** — 表达是匿名的，但可见的
7. **沉淀关系，非积累人脉** — 知己是自然涌现的，不是主动添加的

## 五、核心概念

### 5.1 心物（Seedstone）

**心物是一切交互的起点。它必须是有形的、可被体验的事物，由此生发出来的才是真挚的感情。**

```
                    真实的情感
                       ↑
               用户的反应/感受
                       ↑
    用户 A ──→  [ 锚 点 ]  ←── 用户 B
                    ↑
              现实中的某个事物
```

**没有心物，就没有表达。** 这不是限制，是过滤器——它天然地过滤掉了「为了发而发」的内容。

心物是客观内容，不是引导：
- 心物是「事物」，不是「问题」
- 心物是「触发物」，不是「引导语」
- 心物是「世界给你的」，不是「系统问你的」

### 5.2 感受链（Feeling Chain）

**感受链是 Standby 的核心机制，替代传统的「内容+评论」模式。**

传统社交模型：心物 + 评论
- 我说了什么 → 你来评论 → 我回复你 → 讨论
- 本质是「广播 + 回应」

Standby 模型：心物 + 感受链
- 事物触发我 → 我表达 → 我的表达触发你 → 你表达 → 你的表达触发 ta → ...
- 本质是「链式反应」

**每条表达都是独立的，不是「回复」别人，而是「被触发」的新感受。**

```
心物: [夕阳照片]
  │
  ├─→ A: 「让我想起了外婆家门口的夕阳」
  │     └─→ B: 「我外婆也喜欢看夕阳，但她已经不在了」
  │           └─→ C: 「珍惜还在的人」
  │
  └─→ D: 「美得让人安静」
        └─→ E: 「安静的时候最容易想起不该想的人」
```

**感受链是树状结构，不是线性评论。每个人的感受都可以触发新的分支。**

### 5.3 共鸣（Resonance）

两个人对同一心物产生相似的情感反应。不是点赞，不是评论，而是「我也有过这种感觉」。

**共鸣的真正目的**：不是为了确认你的感受是对的，而是为了帮你理解你的感受可以被怎样应对。

- 共鸣：「有人跟我有同样的感觉」→ 不孤单
- 发现差异：「但 ta 的应对方式跟我不一样」→ 新视角
- 被差异启发：「我也可以试试 ta 的方式」→ 成长

**共鸣是入口，不是终点。通过理解他人来理解自己。**

### 5.4 知己（Confidant）

通过持续共鸣自然沉淀出的关系。不是加好友，不是互关，而是「我们总是不约而同地被同样的东西打动」。

**知己关系的本质是「感受的连接」，不是「身份的连接」。**

- 你不知道 ta 是谁，但你知道 ta 总是跟你有同样的感受
- 你不需要知道 ta 的真名，因为知己关系的价值在于感受的连接
- 你通过感受认识 ta，不是通过名字、长相、职业

### 5.5 匿名身份

所有用户在系统中都是匿名的。没有头像照片、没有真名、没有个人主页链接。只有随机生成的昵称和 emoji。

**匿名不等于真实，但匿名创造了真实的条件。**
- 匿名消除了表演动机（没有观众）
- 匿名消除了社会后果（没有评判）
- 匿名创造了安全空间（可以脆弱）

## 六、用户画像

### 6.1 目标用户

**大城市原子化个体**：
- 一人在大城市工作生活，家人朋友不在身边
- 渴望真挚且真实的交流、沟通和共鸣
- 对现有社交平台失望（表演压力、流量逻辑、虚假内容）
- 有表达欲望但缺乏安全的表达空间

### 6.2 用户核心需求

- 被理解（而非被看见）
- 找到共鸣（而非获取关注）
- 真实连接（而非社交表演）

## 七、商业模式

### 7.1 核心立场

Standby 不采用以下商业模式：
- **不卖用户数据**：用户的一切表达、反应、共鸣关系不对外提供
- **不做广告**：不以流量变现，不存在广告推荐算法
- **不搞会员分级**：不通过付费解锁功能来区分用户
- **不做企业版定制**：不为 B 端客户改变产品对用户的设计原则

### 7.2 变现路径

**路径一：跨设备生态合作**
- SDK/中间件：将核心能力封装为 SDK，设备厂商集成到自有系统中
- 深度合作/联合开发：与特定生态厂商绑定
- 目标生态：小米全生态多终端统一系统（澎湃 OS）

**路径二：基于用户洞察的咨询服务**
- 趋势洞察报告：基于匿名聚合数据的趋势分析
- 决策咨询：为政策制定者、学术机构提供基于真实公众情感的数据支撑

## 八、成功标准

### 8.1 核心原则

留存率、注册量、使用时长是**结果**，不是目标。追逐这些指标会导致讨好用户的设计，背离「真实表达」的产品哲学。

### 8.2 最终检验标准

**当用户自产的心物和平台初始心物在各项表现指标上没有显著差异时——产品成功了。**

用户不再只是消费者，而是和平台一起创造高质量内容。
